# 1️⃣ `app = FastAPI()` — What is this?

```python
from fastapi import FastAPI

app = FastAPI()
```

Here:

* `FastAPI` → class

* `app` → instance (your web application)


# FastAPI Basics

## `app = FastAPI()`

- **`FastAPI`**: Main class to create a FastAPI application.
- **`app`**: Instance of your web application.
- Analogy:
    - `class Car: ...`
    - `my_car = Car()`
    - Here, `app` is like `my_car`—your actual web server object.

```python
from fastapi import FastAPI
app = FastAPI()
```

---

## Running the Server

### `uvicorn main:app --reload`

- **Uvicorn**: ASGI server that runs your FastAPI app.
- **FastAPI does not run the server itself**; Uvicorn does.
- Command breakdown:
    - `uvicorn`: Start the server
    - `main`: Python file (`main.py`)
    - `app`: Variable inside that file
    - `--reload`: Auto-restart on code changes
- `main:app` means: “Find the variable `app` in `main.py`.”

#### Example
```python
# main.py
from fastapi import FastAPI
app = FastAPI()
```

---

### `fastapi dev main.py`

- Newer, developer-friendly command.
- Automatically:
    - Starts Uvicorn
    - Enables reload
    - Handles config for you
- Usage:
    - `fastapi dev main.py` (instead of `uvicorn main:app --reload`)
- Easier for beginners.

---

## Key Differences

| Command                     | What it does                        |
|-----------------------------|-------------------------------------|
| `uvicorn main:app --reload` | Directly runs Uvicorn ASGI server   |
| `fastapi dev main.py`       | Wrapper around Uvicorn, more convenient |

- Both commands run Uvicorn under the hood.
